# Documentation

This directory keeps product-facing documentation. Owner-side follow-up work is
kept at the repository root for easier access.

- `sdk-contract.md`: public SDK surface and compatibility policy.
- `fork-pr-validation.md`: public fork pull request validation model.
- `v1-readiness-checklist.md`: release-readiness evidence checklist.
- [`../NETXT_STEPS.md`](../NETXT_STEPS.md): owner-side follow-up work
  that depends on external accounts, real users, or launch materials.

Stale roadmap and launch-planning notes were removed to keep the public docs
focused on adoption, validation, and concrete owner action.
