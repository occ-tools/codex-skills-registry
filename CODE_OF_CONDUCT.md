# Code of Conduct

## Our standard

This project is for maintainers who want safer, more reusable automation around
Codex Skills, plugin manifests, and MCP server configuration. Participation
should be practical, respectful, and focused on making the tool easier to trust.

Examples of welcome behavior:

- using clear, constructive language
- giving feedback that is specific and actionable
- respecting different experience levels and project constraints
- disclosing security-sensitive findings privately when appropriate

Examples of unacceptable behavior:

- harassment, insults, or personal attacks
- publishing private information without permission
- intentionally derailing technical discussion
- submitting examples with real secrets, tokens, or private service URLs

## Enforcement

Maintainers may edit, hide, or remove comments, issues, pull requests, or other
contributions that violate this standard. Repeated or severe violations may lead
to temporary or permanent bans from project spaces.

## Reporting

Report conduct concerns to the maintainers privately. For security issues, use
the process in `SECURITY.md`.
